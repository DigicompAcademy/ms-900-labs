# Schritt-für-Schritt-Anleitung: Release-Einstellungen in Microsoft 365 ändern

## 1. Voraussetzungen
- Rolle: **Globaler Administrator** erforderlich
- Änderungen können bis zu 24 Stunden dauern, bis sie wirksam werden

## 2. Release-Typen
- **Standard Release**: Updates werden bereitgestellt, sobald sie global verfügbar sind.
- **Gezieltes Release (Targeted Release)**:
  - Für Organisation: Alle Benutzer erhalten Updates früher.
  - Für ausgewählte Benutzer: Nur bestimmte Benutzer erhalten Updates früher.

## 3. Verwaltung im Microsoft 365 Admin Center
1. Melde dich im [Microsoft 365 Admin Center](https://admin.microsoft.com) an.
2. Gehe zu **Einstellungen → Organisationsprofil**.
3. Klicke auf **Release-Einstellungen**.

## 4. Optionen auswählen
- **Standard Release aktivieren**:
  - Wähle „Standard Release“ und speichere.
- **Gezieltes Release für alle aktivieren**:
  - Wähle „Gezieltes Release für alle Benutzer“ und speichere.
- **Gezieltes Release für ausgewählte Benutzer aktivieren**:
  - Wähle „Gezieltes Release für ausgewählte Benutzer“.
  - Benutzer hinzufügen oder hochladen und speichern.

## 5. Nach der Konfiguration
- Änderungen können bis zu 24 Stunden dauern.
- Rückkehr zu Standard Release kann Funktionen entfernen.

## 6. Vorteile gezielter Veröffentlichung
- Frühes Testen neuer Funktionen
- Vorbereitung von Kommunikation und Schulungen
- Compliance- und Sicherheitsprüfungen vor Rollout

## 7. Weiterführende Schritte
- Nutze das **Microsoft 365 Message Center** für Ankündigungen.
- Für lokale Office-Apps siehe: „Overview of update channels for Microsoft 365 Apps“.
